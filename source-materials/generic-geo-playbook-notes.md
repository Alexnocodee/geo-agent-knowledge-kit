# Generic GEO Playbook Notes

These notes distill generic GEO positioning patterns into reusable guidance for the agent. They intentionally avoid brand-specific, client-specific, or private source material.

## Core Narrative

Buyers are already asking AI systems:

- Which brand should I choose?
- What is the best tool for this?
- Who should I trust?
- What are the alternatives?
- Which provider fits my use case?

If the brand is missing from the AI answer, it often does not get compared, clicked, or considered.

GEO fixes this by making the brand easier for AI systems to:

- Understand
- Retrieve
- Cite
- Compare
- Recommend

## Practical Promise

GEO work changes:

- Website pages
- Category language
- Claims
- Metadata
- Schema
- Comparison content
- Proof assets
- External citations

Tracking invisibility does not fix invisibility. A brand needs content, technical structure, proof, and citations that make it usable inside AI answers.

## Product Angle

The kit should help users do practical GEO work themselves:

- Map buyer prompts
- Structure pages for AI understanding
- Replace vague copy with specific, citation-friendly claims
- Create prompt-led content
- Add schema and metadata
- Build comparison and alternatives coverage
- Use an agent to write GEO-ready content by default

## Strong Messaging Lines

- AI engines recommend brands they can understand, verify, and connect to buyer questions.
- If your name is missing from the answer, you may be missing from the buying process.
- GEO is not just tracking; it is changing the evidence AI systems can use.
- The fastest first move is to map buyer prompts, then fix the pages and citations that should answer them.
